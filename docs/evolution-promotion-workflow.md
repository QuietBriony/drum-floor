# Human Promotion Workflow

Promotion is the final gate between a local evolution suggestion and a committed change to `patterns/drum-pattern-frames.json`.

Suggestions are useful hints. They are not source of truth.

## Promotion rule

A suggestion can be promoted only when a human reviewer can explain:

- which score JSON files motivated the change
- which suggestion JSON is being considered
- which pattern frame field would change
- why the change improves the band groove
- how it preserves safety boundaries
- how it can be rolled back

## Required gates

1. Candidate was generated by `python -m drum_floor generate`.
2. Candidate was inspected with `python -m drum_floor inspect`.
3. Candidate was listened to in browser, Ableton, or EP-133 preview routing.
4. Listening score was saved with `python -m drum_floor score`.
5. Suggestion was created with `python -m drum_floor suggest-evolution`.
6. Human reviewer writes a promotion request.
7. A separate PR updates the target pattern frame.

## Allowed promotion targets

Promotion may propose edits to:

- `space`
- `snare_lag_ms`
- `kick_push_ms`
- `ghost_glue`
- `hat_swing`
- `bass_lock`
- `mix_hints`
- fill policy language in docs

Promotion must not directly edit generated candidates, armed candidates, audio, samples, Ableton projects, EP-133 state, or workflow automation.

## Review checklist

- The promoted change is small and reversible.
- The change maps to one or more listening scores.
- The suggestion JSON remains metadata-only.
- The proposed pattern-frame change does not copy songs, samples, recordings, or artist performances.
- The reviewer can describe the musical intent in original language.
- The PR does not touch `live/armed/`.
- The PR does not add dependencies, samples, audio files, or workflow changes.

## Rollback

Rollback is a normal PR that restores the previous pattern frame values and records the reason in the PR body.

Do not rollback by deleting score history or suggestion metadata.

## Non-goals

- No automatic promotion.
- No direct device control.
- No hidden local state as source of truth.
- No generated score or suggestion JSON committed unless it is promoted intentionally as an example.
